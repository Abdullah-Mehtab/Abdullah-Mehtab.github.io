Final Year Project / Security Engineering

Cyber Sentinel - Security Monitoring SIEM.

A final-year security engineering project built to make SIEM-style monitoring more practical for small environments using open-source tools, automated deployment, and alerting.

Project site Back to projects
Wazuh Endpoint/security monitoring backbone
ELK Log search, storage, and dashboards
Suricata Network IDS signal
Small-scale Designed around practical cost constraints

Problem

Security monitoring is often treated like an enterprise-only luxury.

Small teams and labs still need visibility: endpoint events, network alerts, searchable logs, and enough automation that setup does not become the actual project. Cyber Sentinel focused on making that stack understandable, deployable, and demonstrable.

My role

Architecture, integration, automation, and documentation.

I worked across tool selection, deployment flow, alerting behavior, dashboard thinking, and the documentation needed to explain the system to evaluators without hiding behind buzzwords.

Architecture

Open-source components wired into a practical monitoring flow.

01

Collect

Agents and sensors feed endpoint and network activity into the monitoring stack.

02

Detect

Wazuh rules and Suricata alerts surface suspicious behavior and policy-relevant events.

03

Search

ELK-style log handling makes events searchable and easier to investigate.

04

Alert

Email alerting and dashboards make the result visible instead of buried in terminal output.

Security decisions

Useful signal over decorative complexity.

  • Used mature open-source security tools instead of inventing weak detection logic from scratch.
  • Focused on visibility, investigation, and alerting rather than claiming magical prevention.
  • Kept deployment and documentation central so the system could be repeated and evaluated.
  • Designed the project around real constraints: cost, hardware limits, and operator understanding.

Outcome

A portfolio-grade security project with explainable engineering.

The project demonstrates SIEM architecture, IDS integration, alerting, log analysis, automation instincts, and the ability to explain security tooling to technical and non-technical evaluators.
